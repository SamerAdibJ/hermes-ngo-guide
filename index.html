🔑 Step 1 — Get Your DeepSeek API Key

An API key is like a password that lets Hermes talk to DeepSeek's AI models. Here's how to get one.

1

Sign up for a DeepSeek Account

Go to platform.deepseek.com/signup and create an account. You can sign up with your email or Google account.

2

Create an API Key

Once logged in:

1. Go to the API Keys page (left sidebar → API Keys)

2. Click "Create API Key"

3. Give it a name like "Hermes Agent"

4. Copy the key — it looks like sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

⚠️ Copy it now — you won't be able to see it again after you leave the page!

3

Add Credits (Optional)

DeepSeek gives you a small amount of free credits to start. After that, you need to add funds. Go to Billing → Top Up and add $5–10 to start. That will last you weeks of regular use.

DeepSeek pricing (as of June 2026): V4 Flash (default): $0.435 /M input tokens | $0.87 /M output V4 Pro (stronger): $1.74 /M input tokens | $3.48 /M output DeepSeek Chat (V3): $0.14 /M input tokens | $0.28 /M output DeepSeek R1 (reason): $0.55 /M input tokens | $2.19 /M output

💡 Recommendation: Start with DeepSeek V4 Flash. It's fast, capable, and the best value. A full day of NGO work might cost $0.30–0.80.

4

Test Your API Key

You can test that your key works with this curl command in your terminal:

curl https://api.deepseek.com/v1/chat/completions \ -H "Content-Type: application/json" \ -H "Authorization: Bearer sk-your-key-here" \ -d '{"model":"deepseek-chat","messages":[{"role":"user","content":"Hello!"}]}'

You should get a JSON response with DeepSeek's reply. If you see an error, double-check your key.

✅ API key ready! Keep it safe — treat it like a password. Never share it, never paste it into public forums. We'll add it to Hermes in Step 3.

⚡ Step 2 — Install Hermes Agent

Hermes is the orchestrator that uses DeepSeek's AI to take real actions — write files, manage tasks, search the web, and remember your preferences.

1

Install Node.js

Go to nodejs.org and download the LTS version. Run the installer — this is required for Hermes to run. (If you already have Node.js, skip this step.)

2

Install Git

Go to git-scm.com, download, and install.

3

Install Hermes

npx hermes-agent setup

Follow the interactive setup. When it asks about your model provider, you can skip for now — we'll configure DeepSeek in the next step.

4

Check Everything is Installed

hermes doctor

If everything is green, you're ready to connect Hermes to DeepSeek.

🔗 Step 3 — Connect Hermes to DeepSeek

This is where the magic happens — Hermes gets its "brain" from DeepSeek's cloud models.

1

Add Your API Key to Hermes

Run the auth command to securely store your DeepSeek key:

hermes auth add deepseek

Paste your API key when prompted. Hermes stores it securely in its config.

Alternatively, you can set it directly in the .env file:

DEEPSEEK_API_KEY=sk-your-key-here
2

Set DeepSeek as Your Provider

Use the interactive model picker:

hermes model

Select DeepSeek as your provider and choose your preferred model. Or set it directly in config:

hermes config edit

And add or edit these lines:

model: default: deepseek-chat # or deepseek-v4-flash, deepseek-v4-pro, deepseek-reasoner provider: deepseek
3

Test the Connection

hermes chat -q "Hello! Who are you and what can you do?"

If you get a response from DeepSeek — congratulations! 🎉 You now have Hermes running on cloud AI.

⚠️ Important note on model aliases: DeepSeek occasionally remaps the model alias deepseek-chat (V3) to newer models (V4 Flash) on their backend. This means you could be billed at V4 Flash rates (~3x more) while your config still says deepseek-chat. To be safe and transparent, set the explicit model ID:
hermes config set model.default deepseek-v4-flash

If you want the absolute best model (stronger reasoning):

hermes config set model.default deepseek-v4-pro

Check your actual usage on DeepSeek's dashboard to verify which model was served.

🧠 Holographic Memory Setup

This is what makes Hermes remember you across sessions — your name, preferences, past conversations, and learned facts. Runs entirely on your local machine.

📝

Built-in Memory (Default)

Saves key facts as text at the top of the system prompt. Simple, works well, but limited to ~2,200 characters. Good enough for preferences and quick notes.

🧬

Holographic Memory (Advanced)

Uses a local SQLite database with entity resolution, trust scoring, and semantic search. No character limit, remembers structured facts, and can reason across multiple entities. Runs 100% locally.

Enable Holographic Memory

1

Check Current Memory Status

hermes memory status
2

Configure Holographic Memory Provider

Edit your config (hermes config edit) and add or update:

memory: memory_enabled: true user_profile_enabled: true provider: holographic
3

Restart and Test

Exit and restart Hermes:

hermes # Then inside the session, test it: "Remember that my name is [your name] and I work for FES NGO"

In the next session, Hermes will remember who you are.

How Holographic Memory Works

Unlike the built-in memory (which stores plain text notes), Holographic Memory stores structured facts with entity resolution — it knows that "Samer at FES" and "the person who asked about the report" are the same entity. It uses trust scoring to learn which facts are reliable and which are outdated. And it can reason across entities: "What does Samer know about WhatsApp?"

Feature Built-in Memory Holographic Memory
Storage 2,200 character limit Unlimited SQLite database
Search Keyword match Semantic + keyword + entity resolution
Persistence Across sessions Across sessions + profiles
Confidence All facts equal Trust scoring per fact
Offline ✅ Yes ✅ Yes (local SQLite)
Provider Built-in (always on) holographic (configurable)
🔑 Key point: With Holographic Memory, Hermes can remember everything about your work — donor preferences, reporting templates, meeting notes from weeks ago, communication styles for each partner, and recurring tasks. It's like having an assistant who never forgets. And the memory runs locally on your machine — it never touches the cloud, not DeepSeek or any provider.

🎯 Which Provider & Model Should You Use?

DeepSeek offers several models. Here's how they compare and which to pick.

✅ Providers That Support DeepSeek

Hermes natively supports DeepSeek Direct (api.deepseek.com) as a built-in provider. You can also use DeepSeek models through other providers that offer them:

Provider Endpoint Setup Notes
🌐 DeepSeek Direct api.deepseek.com Sign up → API key → hermes auth add deepseek Recommended. Lowest cost, direct access, fastest.
🎯 OpenRouter openrouter.ai/api/v1 Sign up → Create API key → Configure in Hermes Accesses DeepSeek + many other models via one key. Great for testing different models.
🤖 Together AI api.together.xyz/v1 Sign up → API key → Custom endpoint config Hosts DeepSeek models with good speed. Also hosts open-source models.
🏆 Recommendation for most users: DeepSeek Direct — sign up at platform.deepseek.com, get an API key, and set Hermes to use deepseek-v4-flash. Best value for NGO work.

DeepSeek Model Comparison

Model Config Name Input / 1M tokens Output / 1M tokens Best For

deepseek-chat (V3, legacy) $0.14 $0.28 Cheapest option. Good for simple tasks. Note: DeepSeek may silently upgrade this alias to V4 Flash.

🔧 Troubleshooting

Common issues with cloud-based setup and how to fix them.

🔑

"Authentication failed" or "Invalid API key"

Your API key is incorrect or expired. Generate a new one at platform.deepseek.com/api-keys and update it with hermes auth add deepseek.

💰

"Insufficient balance" error

You've run out of credits. Top up at DeepSeek Platform → Billing. $5 usually lasts a month for daily NGO use.

🌐

"Connection refused" / Network error

Check your internet connection. The API endpoint should be https://api.deepseek.com/v1. If you're behind a corporate firewall or VPN, you may need to whitelist api.deepseek.com.

💸

Costs higher than expected

DeepSeek may have silently upgraded your deepseek-chat alias to a more expensive model. Set the explicit model: hermes config set model.default deepseek-v4-flash. Check usage at platform.deepseek.com/usage.

🐢

Slow responses

Usually a network issue. Check your connection speed. Sometimes DeepSeek's servers are under load — wait a moment and try again. If persistent, try a different model.

❓

"Provider not available" in Hermes

Run hermes doctor to check your config. Make sure provider: deepseek is set correctly in config.yaml and your API key is saved.
