🦙 Step 1 — Install Ollama & Download a Model

Ollama is the easiest way to run local LLMs on Windows. It handles GPU acceleration, model management, and serves an OpenAI-compatible API automatically.

1

Download & Install Ollama

Go to ollama.com/download and download the Windows installer. Run it — it installs like any normal program. After installation, Ollama runs in the system tray.

Verify it works — open a terminal (Command Prompt or PowerShell): ollama --version
2

Download a Model

Pick a model from the list below. The first download takes a few minutes depending on model size. After that, it's cached locally.

# For the fastest experience — fits entirely in your RTX 5080 16GB: ollama pull qwen2.5:14b-instruct-q4_K_M # For better quality and still fast: ollama pull qwen2.5:32b-instruct-q4_K_M # For maximum intelligence (largest, uses GPU + CPU): ollama pull command-r:35b-v01-q4_K_M

💡 Recommendation: Start with qwen2.5:14b to test, then upgrade to 32b for better results on your RTX 5080.

3

Confirm the Model is Running

Test it with a quick prompt:

ollama run qwen2.5:14b-instruct-q4_K_M "What is Hermes Agent?"

You should get a response. Press Ctrl+D to exit when done.

4

Keep the API Server Running

Ollama automatically runs a background server at http://localhost:11434 once installed. This is the endpoint Hermes will talk to.

# Test the API directly: curl http://localhost:11434/v1/chat/completions -d '{"model":"qwen2.5:14b","messages":[{"role":"user","content":"Hello"}]}'
✅ Ollama is now running! The OpenAI-compatible server at localhost:11434 is ready. No internet needed, no API key required.

⚡ Step 2 — Install Hermes Agent

Hermes is the orchestrator that uses the local LLM to take real actions.

1

Install Node.js

Go to nodejs.org and download the LTS version. Run the installer — this is required for Hermes to run.

2

Install Git

Go to git-scm.com, download, and install.

3

Install Hermes

npx hermes-agent setup

Follow the interactive setup. When it asks about a model provider, choose "Custom endpoint" — we'll configure it in the next step.

4

Check it Works

hermes doctor

If everything is green, you're ready to connect Hermes to your local model.

🔗 Step 3 — Connect Hermes to Your Local LLM

This is the magic step — Hermes sends requests to Ollama instead of a cloud API.

1

Open Hermes Configuration

hermes config edit

This opens the config file in your default text editor.

2

Configure the Local Provider

Add or edit these lines under the model section:

model: default: qwen2.5:14b provider: custom base_url: http://localhost:11434/v1 api_key: not-needed
3

Or use the setup wizard

Alternatively, run the model setup from the terminal:

hermes model

Select "Custom endpoint" → enter http://localhost:11434/v1 → set model name to qwen2.5:14b → leave API key blank.

4

Test the Connection

hermes chat -q "Hello, who are you?"

If you get a response from your local model — congratulations! You now have a fully offline AI agent. 🎉

⚠️ Important: For the 32B or larger models that don't fully fit in 16GB VRAM, Ollama automatically offloads some layers to CPU + RAM. This is normal. Speed will still be usable — a few seconds per response instead of sub-second. To force more GPU layers, you can set OLLAMA_NUM_GPU=50 or similar in environment variables.

🧠 Holographic Memory Setup

This is what makes Hermes remember you across sessions — your name, preferences, past conversations, and learned facts.

📝

Built-in Memory (Default)

Saves key facts as text at the top of the system prompt. Simple, works well, but limited to ~2,200 characters. Good enough for preferences and quick notes.

🧬

Holographic Memory (Advanced)

Uses a local SQLite database with entity resolution, trust scoring, and semantic search. No character limit, remembers structured facts, and can reason across multiple entities. Runs 100% offline.

Enable Holographic Memory

1

Check Current Memory Status

hermes memory status
2

Configure Holographic Memory Provider

Edit your config (hermes config edit) and add or update:

memory: memory_enabled: true user_profile_enabled: true provider: holographic
3

Restart and Test

Exit and restart Hermes:

hermes # Then inside the session, test it: "Remember that my name is [your name] and I work for FES NGO"

In the next session, Hermes will remember who you are.

How Holographic Memory Works

Unlike the built-in memory (which stores plain text notes), Holographic Memory stores structured facts with entity resolution — it knows that "Samer at FES" and "the person who asked about the report" are the same entity. It uses trust scoring to learn which facts are reliable and which are outdated. And it can reason across entities: "What does Samer know about WhatsApp?"

Feature Built-in Memory Holographic Memory
Storage 2,200 character limit Unlimited SQLite database
Search Keyword match Semantic + keyword + entity resolution
Persistence Across sessions Across sessions + profiles
Confidence All facts equal Trust scoring per fact
Offline ✅ Yes ✅ Yes (local SQLite)
Provider Built-in (always on) holographic (configurable)
🔑 Key point for his use case: With Holographic Memory, Hermes can remember everything about his NGO work — donor preferences, reporting templates, meeting notes from weeks ago, communication styles for each partner, and recurring tasks. It's like having an assistant who never forgets.

🔧 Troubleshooting

Common issues and how to fix them.

❌

Hermes says "Provider not available"

Check that base_url in config.yaml points to http://localhost:11434/v1. Make sure Ollama is running. Test with curl http://localhost:11434.

🐢

Responses are very slow

Check how many GPU layers are used. Run ollama ps to see. If the model is too big, try a smaller one (14B instead of 32B). Close other GPU apps (browser, games).

💾

Out of VRAM error

The model is too large for 16GB. Switch to a smaller quant (Q3 or Q2) or a smaller model. Qwen 2.5 14B Q4_K_M will always fit.

🔄

Model not responding

Restart Ollama: right-click tray icon → Quit, then relaunch. Or run ollama serve in a terminal to see logs.
